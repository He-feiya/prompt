# 模型为什么会"忽略"Prompt中的部分指令讲解脚本

**时长：约5-6分钟 | 字数：约1000字**

---

## 【开场白】（约30秒）

大家好！今天我们来讨论一个非常常见但令人沮丧的问题——**为什么AI有时会忽视你Prompt中的某些指令？**

你明明写了要求，但AI就是不听。是AI在反抗吗？不是。这背后有具体的技术原因。

---

## 【问题现象】（约1分钟）

首先，让我们看看这个问题的具体表现。

你写的Prompt：
```
请用中文回答，字数200字以内，使用简洁风格，
不要使用技术术语，提供3个具体例子。
```

AI的回答可能会：
- 忽视字数限制，写了500字
- 完全用技术术语
- 只给了1个例子
- 用了复杂的语言而非简洁风格

为什么会这样？这不是AI在"调皮"，而是有深层的技术原因。

---

## 【根本原因分析】（约1.5分钟）

为什么AI会忽略指令？有几个主要原因。

**原因一：注意力机制的限制**

AI使用"注意力机制"来处理信息。在长Prompt中，AI对后面的指令可能关注度较低。这就像人类在看长文档时，容易忽视后面的细节。

特别是当Prompt很长时，前面的内容会"占用"大部分注意力，后面的指令就被相对忽视。

**原因二：指令冲突**

有时候，多个指令之间可能有冲突。比如：

```
请用幽默的语调回答这个医学问题。
但要保持专业和严肃。
```

这两个要求是相矛盾的。AI会优先遵循"更突出"的指令，忽视另一个。

**原因三：指令的优先级不清**

当有多个指令时，AI不知道哪个是最重要的。它可能会权衡各个指令，结果都不完全满足。

**原因四：训练数据中的偏好**

AI是在大量文本数据上训练的。如果训练数据中，某种风格的文本更常见，AI就更倾向于生成那种风格，甚至忽视相反的指令。

比如，如果AI在专业文本上训练较多，它可能倾向于用正式语言，即使你要求非正式。

**原因五：复杂指令难以执行**

某些指令本身就很难同时满足。比如：

```
写一个故事，但字数不超过50字，
包含开头、中间、结尾、三个角色、
完整的情节和深刻的主题。
```

这些要求几乎不可能同时满足。AI会放弃某些要求。

---

## 【为什么长Prompt特别容易被忽略】（约1分钟）

有一个特别的问题值得单独讨论：为什么长Prompt的后面内容更容易被忽略？

这涉及到AI的"位置编码"和"注意力衰减"问题。简单说：
- AI在处理序列数据时，后面的位置的权重会逐渐降低
- 这被称为"注意力衰减"
- 结果就是，Prompt越长，后面的指令越容易被忽视

所以，如果你把最重要的指令放在Prompt的最后，反而更容易被忽略！

---

## 【如何确保指令被遵循】（约1.5分钟）

怎样避免指令被忽略？

**技巧一：重复重要指令**

重复最关键的要求，这样即使AI忽略了一次，也可能在重复中注意到。

```
#关键要求#
【重要】字数必须在200字以内。
【重要】不要使用技术术语。

回答：[内容]

确保字数不超过200字。
```

**技巧二：优先级明确**

使用清晰的优先级标记：

```
【最高优先级】用中文回答
【高优先级】字数不超过200字
【中优先级】使用简洁风格
【低优先级】如果可能，提供例子
```

**技巧三：指令前置**

把最重要的指令放在Prompt的开头，而不是结尾。

不好：[长内容]……最后加上一句"不要忘记……"
好：开头明确说"最重要的是……"

**技巧四：分解复杂指令**

如果指令太复杂，分成多个简单的指令：

不好：写一个故事，字数50字，包含所有这些元素……
好：
```
第一步：写一个故事大纲
第二步：把故事扩展到50字
第三步：检查是否包含所有必要元素
```

**技巧五：使用强制格式**

指定一个强制格式能帮助确保指令被遵循：

```
请用以下格式回答：

【开头】
【主要内容】（不超过150字）
【结论】
【例子】
```

---

## 【验证指令遵循度】（约1分钟）

如何判断指令是否被遵循？

添加一个验证环节：

```
生成回答后，请检查：
□ 是否用中文？
□ 字数是否在200以内？
□ 是否使用了简洁风格？
□ 是否避免了技术术语？
□ 是否包含3个例子？

如果有任何一项不符合，请重新生成。
```

---

## 【总结】（约30秒）

**AI忽视指令的原因包括注意力限制、指令冲突、优先级不清等。**

解决方法：
- 把重要指令放在开头并重复
- 明确优先级
- 分解复杂指令
- 使用强制格式
- 添加验证环节

记住：指令越清晰、越简短、越优先级明确，被遵循的可能性就越大。
